# WebSocket Event Contract

This document defines the canonical JSON schema for all Mercator WebSocket application events.

All event fields are snake_case.

## Message Envelope

All application events use the same envelope:

```json
{
  "event_type": "string",
  "timestamp": "2026-05-08T12:34:56.000000",
  "payload": {}
}
```

- event_type: event name (snake_case)
- timestamp: UTC ISO8601 timestamp generated by backend
- payload: event-specific object

## Event Types

### new_listing

```json
{
  "event_type": "new_listing",
  "timestamp": "2026-05-08T12:34:56.000000",
  "payload": {
    "listing_id": "string",
    "seller_wallet": "string",
    "seller_name": "string",
    "price_usdc": 0,
    "insight_preview": "string",
    "source_type": "string",
    "ipfs_cid": "string",
    "listing_tx_id": "string",
    "reputation_score": 0
  }
}
```

Notes:
- insight_preview is the first 100 characters of the insight text, not full content.

## Infrastructure Keepalive Event

Heartbeat pings are transport-level keepalive events and are not part of application business events.

Current implementation uses:

```json
{
  "event_type": "ping",
  "timestamp": "2026-05-08T12:34:56.000000",
  "payload": {}
}
```

Clients should reply with:

```json
{"type":"pong"}
```
