V2 Phase 5
1. Build the seller-facing React UI for uploading trading insights in one seamless flow.
Use the VibeKit starter template to create a clean, single-page form where a human trader enters insight text, sets a USDC price, provides their wallet address, and clicks one button to upload everything; connect it directly to the backend so the UI triggers IPFS upload followed by on-chain ASA creation; add real-time feedback (loading states, success messages, transaction link) so any trader can list an insight in under 30 seconds.






To-Do:
1. Navigate into the frontend directory of your AlgoKit project.
In the terminal run cd frontend so you are working inside the official React + Vite boilerplate that was generated in Point 1.
2. Install additional UI and HTTP libraries needed for the form.
Run npm install axios react-hot-toast @heroicons/react — these give you clean form handling, toast notifications for feedback, and simple icons without adding heavy dependencies.
3. Create the main seller form component.
Inside frontend/src/ create a new file SellInsight.tsx (or replace App.tsx content if you prefer a single-page app) and set up a clean React functional component with Tailwind classes for a professional single-page layout.
4. Build the form fields exactly as specified.
Add a large <textarea> for the trading insight text, two <input> fields (one type="number" for USDC price with step="0.000001", one type="text" for seller wallet address), and a prominent “List Insight on Algorand” button — keep the UI minimal and mobile-friendly.
5. Add real-time state management for loading, success, and error.
Use React useState hooks to track isLoading, successTxId, and errorMessage so the UI can show a spinner during submission, a success message with the transaction link, or a clear error.
6. Create the submit handler that connects to the backend.
On button click, collect form values, call axios.post('http://localhost:8000/list', { insight_text, price, seller_wallet }) (using the FastAPI endpoint you will build later), and handle the response to update the UI states.
7. Integrate the IPFS + on-chain flow through the backend response.
After the backend returns success, display the transaction ID as a clickable link to https://testnet.explorer.algorand.org/tx/{txId} so the seller sees instant confirmation that the insight is now live on-chain.
8. Add loading spinner and disabled state during submission.
While isLoading is true, disable the button, show a spinning Heroicon, and change the button text to “Uploading to IPFS & Algorand…” so the user knows the process is happening and cannot double-submit.
9. Implement success and error toast notifications.
Use react-hot-toast to show a green success toast (“Insight listed successfully! View on explorer”) with the link, or a red error toast with the exact message from the backend, so feedback is instant and user-friendly.
10. Wire the component into the app and run it locally.
Import and render <SellInsight /> as the main content in App.tsx, then run npm run dev in the frontend folder — open http://localhost:5173 (or the port shown) and test the full form: enter sample data → click upload → watch loading → see success message + live TestNet transaction link in under 30 seconds.
