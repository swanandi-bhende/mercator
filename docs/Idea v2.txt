Idea v2
1. Define the problem that you are solving. (e.g., agent to agent communication, reg-tech in finance, privacy in e-commerce, yield management).
Autonomous AI agents cannot easily discover, evaluate, and buy fresh trading insights (stock predictions, candlestick patterns, sentiment reports) from real human experts because platforms are siloed, payments are slow or expensive, and there is no simple way for agents to pay and get data instantly.
2. Describe your project in less than 200 words.
Mercator is a simple decentralized marketplace on Algorand where human traders upload real trading insights and AI agents can find, reason about, and buy them instantly.
Experienced traders (sellers) upload text insights with price and wallet address. AI buyer agents use natural language to search, evaluate matches, and pay automatically. Payments happen through x402 micropayments in USDC that settle on Algorand. All insights come only from real human sellers – no AI-generated data.
The platform uses on-chain listings for trust, IPFS for storing content, and reputation scores so agents avoid low-quality sellers. It is built as an open-source module for the Agentic Commerce track.
For Round 2, the core feature is the complete end-to-end micropayment flow: AI agent discovers insight → evaluates → triggers x402 payment → receives content link instantly. Buyers (AI trading agents) get better alpha for NSE or other markets. Sellers earn small payments passively. This solves discovery and payment problems so AI agents can trade smarter with real expert data.
3. Describe your implementation approach.
I am using the x402 SDK to let AI agents built with LangChain and Google Gemini make instant micropayments for insights, with Algorand as the settlement layer using USDC.
I am building the core smart-contract logic using AlgoKit (replacing deprecated PyTeal) for on-chain listings as ASAs, basic escrow for paid content unlock, and simple reputation scoring. Insights files are stored on IPFS with hashes on-chain. The buyer agent uses semantic search to find matches, does step-by-step reasoning, and triggers x402 payment when the user approves. After payment, the agent automatically receives the content link.
The full single-user flow is implemented end-to-end: agent query → on-chain lookup → x402 micropayment → transaction confirmation → instant data access. Everything is direct Algorand integration, fully functional, open-source, and ready for testing on Testnet. This meets Round 2 requirements with a working product focused on the micropayment + agent integration core.
