V2 Phase 3
1. Deploy the smart contracts to Algorand Testnet and configure all necessary on-chain identifiers.
Use “algokit deploy” to push both the ASA listing contract and the reputation/escrow app to Testnet in a single command; capture the exact App ID, ASA ID, and escrow address; store them securely in a .env file along with Testnet node URLs; run a quick manual transaction test to verify creation and basic state updates so every later step can reference live on-chain components without any hard-coded values.




To-Do:
1. Create a dedicated TestNet deployer wallet.
Run algokit wallet new --name mercator-deployer in your terminal; copy the generated address and the full 25-word mnemonic — this wallet will be used exclusively for deployment and testing on Testnet.
2. Fund the deployer wallet with TestNet ALGO using the official dispenser.
Run algokit dispenser login (if not already authenticated), then execute algokit dispenser fund --receiver <your-deployer-address> --amount 5000000 --whole-units to receive approximately 5 ALGO (sufficient for multiple contract deployments and test transactions).
3. Create a TestNet-specific environment file.
Copy your existing .env file to .env.testnet in the project root — this keeps TestNet configuration cleanly separated from localnet settings.
4. Add the official Algorand TestNet node URLs to .env.testnet.
Open .env.testnet and add: ALGOD_SERVER=https://testnet-api.algonode.cloud, INDEXER_SERVER=https://testnet-idx.algonode.cloud, ALGOD_PORT=443, and ALGOD_TOKEN="" (these are the reliable public AlgoNode endpoints used by most AlgoKit projects).
5. Add the deployer account details to .env.testnet.
Paste the mnemonic into DEPLOYER_MNEMONIC="your 25-word mnemonic here" and the address into DEPLOYER_ADDRESS=your-address-here so the deploy command can automatically sign all transactions.
6. Deploy all smart contracts to TestNet in a single command.
Run algokit project deploy testnet from the project root — it will automatically detect your three contracts (InsightListing, Escrow, Reputation), use the .env.testnet settings, and push everything live in one go.
7. Capture every on-chain identifier from the deployment output.
Immediately after the command finishes, copy the exact values shown in the terminal: InsightListing App ID, Escrow App ID, Reputation App ID, any ASA ID created during deployment, and the escrow contract address.
8. Store all identifiers securely in the main .env file.
Open your main .env and add clear variables: INSIGHT_LISTING_APP_ID=123456789, ESCROW_APP_ID=..., REPUTATION_APP_ID=..., SAMPLE_ASA_ID=..., ESCROW_ADDRESS=ABC... using the exact values from the output.
9. Configure TestNet node URLs in the main .env for runtime use.
Add or update ALGOD_URL=https://testnet-api.algonode.cloud and INDEXER_URL=https://testnet-idx.algonode.cloud in the main .env so the FastAPI backend and LangChain agent can connect directly to the live network.
10. Run a quick manual transaction test to verify creation and state updates.
Use the generated client or a short Python snippet (e.g., from the contract’s application.json) to call create_listing on the InsightListing app with sample price, seller wallet, and IPFS hash; confirm the transaction succeeds on the block explorer, state updates correctly, escrow logic responds, and reputation score initializes — this proves the live on-chain components are fully functional and ready for the rest of the roadmap.
